# 服务参数定义与命令下发功能整改方案

## 一、问题分析

### 问题 1：服务参数定义功能不足（Products.vue）

当前在产品编辑 → 服务标签 → 添加服务中，参数定义仅有以下字段：
- `name`（参数名）
- `type`（类型：string/int/float/bool）
- `required`（是否必填）
- `default`（默认值）
- `description`（描述）

**缺少的功能**：
1. **可选值（常用值）**：用户无法预设常用值列表，在下发页面点击输入框可弹出气泡选择
2. **是否允许手动输入**：无法控制是否允许用户自由输入，还是只能从可选值中选择
3. **范围限制**：数字类型参数缺少最小值/最大值范围限制
4. **正则限制**：字符串类型缺少正则表达式验证规则
5. **默认值类型处理**：当前 default 存为字符串，没有根据 type 做类型转换

### 问题 2：命令下发页面参数输入框无法输入（Commands.vue）

**问题根因分析**：
1. **响应式对象动态 key 问题**：`paramForm` 是 `reactive({})`，通过 `paramForm[param.name] = xxx` 动态赋值，在 Vue 3 中理论上是响应的，但在首次渲染时如果 key 不存在，可能导致输入框初始无法输入
2. **数字类型默认值问题**：`default` 在 Products.vue 中存为字符串，Commands.vue 中读出来也是字符串，配合 `v-model.number` 会造成类型不匹配的警告
3. **缺少参数编辑**：Products.vue 中只有「新增服务」功能，没有「编辑已有服务参数」的功能，已添加的服务无法修改参数定义

---

## 二、整改方案

### 2.1 参数定义数据结构升级

#### 2.1.1 后端 Schema（schemas/__init__.py）

**当前参数结构（数组形式）**：
```json
{
  "name": "brightness",
  "type": "int",
  "required": false,
  "default": "",
  "description": ""
}
```

**升级后的参数结构**：
```json
{
  "name": "brightness",
  "type": "int",
  "required": false,
  "default": 100,
  "description": "灯光亮度",
  "options": [10, 50, 100],
  "allow_custom": true,
  "min": 0,
  "max": 100,
  "pattern": "^[a-zA-Z0-9]+$",
  "step": 1
}
```

**字段说明**：
| 字段 | 类型 | 适用类型 | 说明 |
|------|------|----------|------|
| `name` | string | 全部 | 参数名 |
| `type` | string | 全部 | string/int/float/bool |
| `required` | bool | 全部 | 是否必填 |
| `default` | any | 全部 | 默认值（与 type 匹配） |
| `description` | string | 全部 | 参数描述 |
| `options` | array | string/int/float | 可选值列表（预设常用值），如 [10, 50, 100] |
| `allow_custom` | bool | string/int/float | 是否允许手动输入自定义值（false 则只能从 options 选） |
| `min` | number | int/float | 最小值（数字范围限制） |
| `max` | number | int/float | 最大值（数字范围限制） |
| `step` | number | int/float | 步进值（默认 1 或 0.01） |
| `pattern` | string | string | 正则表达式（默认 `^[a-zA-Z0-9_]+$`） |

**后端改动**：无需修改数据库字段（因为 input_params 是 JSON 字段），仅需更新文档注释说明新结构。

#### 2.1.2 前端 Products.vue 参数定义表单升级

修改 Products.vue 中 `addServiceParam()` 函数和参数编辑表单：

**改动点 A：`addServiceParam()` 初始化新字段**
```javascript
// 旧：
serviceParams.value.push({
  name: '', type: 'string', required: false, default: '', description: ''
})

// 新：
serviceParams.value.push({
  name: '',
  type: 'string',
  required: false,
  default: '',
  description: '',
  options: [],          // 新增：可选值列表
  allow_custom: true,   // 新增：是否允许手动输入
  min: '',              // 新增：数字最小值
  max: '',              // 新增：数字最大值
  step: '',             // 新增：数字步进
  pattern: '^[a-zA-Z0-9_]+$'  // 新增：字符串正则
})
```

**改动点 B：参数编辑表单扩展（根据 type 动态显示字段）**

在现有 5 个字段基础上，新增以下表单项目：
- **可选值（options）**：多标签输入框（el-input + tag 形式），用户输入逗号分隔或回车添加多个值
- **允许手动输入（allow_custom）**：el-switch 开关（默认开启）
- **最小值（min）**：仅 type=int/float 时显示
- **最大值（max）**：仅 type=int/float 时显示
- **步进值（step）**：仅 type=int/float 时显示
- **正则表达式（pattern）**：仅 type=string 时显示，默认值 `^[a-zA-Z0-9_]+$`

**改动点 C：参数默认值类型转换**
- 当 type=int 或 float 时，default 转为 number 类型存储（而非字符串）
- 当 type=bool 时，default 用 el-switch（true/false）

**改动点 D：编辑已有服务的功能**
- 在服务列表表格中增加「编辑」按钮
- 点击「编辑」时，将服务数据加载到 serviceForm 和 serviceParams 中，复用新增服务的对话框进行编辑
- 提交时调用 updateService API（需要后端提供）或 addService 的变种

### 2.2 命令下发页面升级（Commands.vue）

#### 2.2.1 参数输入框修复（解决无法输入的问题）

**核心修复：确保 paramForm 的响应式和默认值初始化正确**

1. **使用 shallowReactive + 显式 key 预注册**
   - 将 `paramForm` 改成在 `loadServiceParams()` 时先清空再重新赋值，确保所有 key 都在对象中

2. **默认值类型严格匹配**
   - type=int/float：`paramForm[name] = Number(default) || 0`
   - type=bool：`paramForm[name] = default === true || default === 'true'`
   - type=string：`paramForm[name] = String(default || '')`

3. **输入组件改进**
   - string 类型：如果设置了 options，则用 el-select + allow-create（可选择可输入）
   - int/float 类型：如果设置了 options，则用 el-select（可选值）+ 允许自定义输入
   - int/float 类型：应用 min/max/step 属性到输入框上
   - bool 类型：继续使用 el-switch

#### 2.2.2 根据新的参数字段渲染合适的输入控件

新的参数渲染逻辑（用伪代码表达）：

```
for each param in currentServiceParams:
  if param.type == 'bool':
    <el-switch v-model="paramForm[param.name]" />
  else if param.type == 'string':
    if param.options 有值:
      <el-select
        v-model="paramForm[param.name]"
        :allow-create="param.allow_custom"
        filterable
      >
        <el-option v-for="opt in param.options" :value="opt" :label="opt" />
      </el-select>
    else:
      <el-input
        v-model="paramForm[param.name]"
        :placeholder="param.description || param.name"
      />
      (若 param.pattern 有设置，失焦时用正则验证)
  else if param.type == 'int' or param.type == 'float':
    if param.options 有值 and !param.allow_custom:
      <el-select v-model="paramForm[param.name]">
        <el-option v-for="opt in param.options" :value="Number(opt)" :label="opt" />
      </el-select>
    else:
      <el-input-number
        v-model="paramForm[param.name]"
        :min="Number(param.min) || undefined"
        :max="Number(param.max) || undefined"
        :step="Number(param.step) || (param.type === 'int' ? 1 : 0.01)"
      />
      (若有 options，可在旁边放一个快捷选择气泡/按钮)
```

#### 2.2.3 参数提交前的验证增强

发送命令前，除了校验 required，还增加：
- 数字范围校验：`min <= value <= max`
- 字符串正则校验：`new RegExp(pattern).test(value)`
- allow_custom=false 时校验：值必须在 options 列表中

## 四、具体实施步骤

### Step 1：升级 Products.vue 参数定义表单

1. 修改 `addServiceParam()`，添加新字段初始化
2. 扩展参数表单 UI（在现有 `<div v-for="(param, index) in serviceParams">` 中新增以下表单项）：
   - `options`：用 el-input + 动态 tag 方式（类似 el-select 的 tag 模式），或简单用逗号分隔字符串，提交时 split 成数组
   - `allow_custom`：el-switch
   - `min` / `max` / `step`：条件显示（type 为 int/float 时）
   - `pattern`：条件显示（type 为 string 时），带默认值 `^[a-zA-Z0-9_]+$`
3. 提交时，根据 `type` 将 `default` 做类型转换（string → number/bool）
4. 新增「编辑服务」功能：在服务列表表格中增加「编辑」按钮，点击时将服务加载到对话框，编辑后提交更新

### Step 2：修复 Commands.vue 参数输入

1. 在 `loadServiceParams()` 中，确保 `paramForm` 的所有 key 都被显式赋值（先用 Object.keys 清空，再逐一赋值）
2. 将默认值做严格类型转换：int/float 用 `Number()`，bool 用 `=== true`，string 用 `String()`
3. 重写参数渲染逻辑，按上述 2.2.2 的规则选择合适的组件
4. 提交前增加范围、正则、options 校验

### Step 3：端到端测试

1. 登录系统，进入产品管理
2. 编辑一个产品，添加服务并定义参数（测试各种 type 的字段是否正确显示）
3. 设置默认值、可选值、范围等，保存
4. 进入命令下发页面，选择设备和刚设置的服务
5. 验证参数输入框是否正常渲染并可输入
6. 验证默认值是否自动填充
7. 验证可选值是否弹出选择
8. 验证范围/正则限制是否生效
9. 点击发送命令，验证设备模拟器是否收到参数

## 六、风险点

| 风险 | 应对方案 |
|------|----------|
| `paramForm` 动态 key 响应式失效 | 改用 `ref({})` + 每次 `loadServiceParams` 时整体替换对象 |
| 默认值字符串转数字时 NaN | 使用 `param.default !== '' ? Number(param.default) : ''`，避免 0 被误判 |
| 正则表达式写得不对导致完全无法输入 | 在 pattern 输入框旁加「测试」按钮，或用 `try-catch` 包裹正则验证 |
| el-select + allow-create 的使用复杂 | 先用简单的「el-select + 无 options 时退化为 el-input」策略 |
| allow_custom=false 但 options 为空时无法输入 | 自动降级为 allow_custom=true，并给出提示 |
| min > max 配置错误 | 在 Products.vue 中添加 min <= max 前端校验 |

## 二号审核修复（v1.1）

### 审核发现的问题及修复建议

#### 【严重】问题1：bool类型默认值处理逻辑错误

**位置**：2.2.1 默认值类型严格匹配 - bool

**当前代码**：
```javascript
paramForm[name] = default === true || default === 'true'
```

**问题分析**：
- 当后端存储的 true 是 JSON boolean 时，`default === 'true'`（字符串比较）永远是 false
- 当后端存储的 true 是字符串 'true' 时，比较才有效
- 如果后端存储为数字 1 或字符串 '1'，也无法正确处理

**修复建议**：
```javascript
// 统一转换为 boolean
paramForm[name] = default === true || default === 'true' || default === 'True' || default === 1 || default === '1'
```
或更健壮的写法：
```javascript
paramForm[name] = [true, 'true', 'True', 1, '1'].includes(default)
```

#### 【严重】问题2：allow_custom 与 options 共存的渲染逻辑缺失

**位置**：2.2.2 参数渲染逻辑

**问题分析**：
伪代码只处理了 `options 有值` 的情况，没有说明当 `options 有值 AND allow_custom=true` 时该如何渲染。

**遗漏场景**：
- 用户定义了 [10, 50, 100] 作为常用值
- 同时 allow_custom=true 允许用户输入自定义值
- 此时应该用 `el-select` + `allow-create` + `filterable`

**修复建议**：
```javascript
else if param.type == 'string':
  if param.options && param.options.length > 0:
    <el-select
      v-model="paramForm[param.name]"
      :allow-create="param.allow_custom !== false"
      :filterable="param.allow_custom !== false"
      default-first-option
    >
      <el-option v-for="opt in param.options" :value="opt" :label="opt" />
    </el-select>
  else:
    // 无 options 的情况...
```

对于 int/float 类型：
```javascript
else if param.type == 'int' or param.type == 'float':
  if param.options && param.options.length > 0:
    if param.allow_custom === false:
      // 只允许从 options 选择
      <el-select v-model="paramForm[param.name]">
        <el-option v-for="opt in param.options" :value="Number(opt)" :label="opt" />
      </el-select>
    else:
      // 允许选择也允许自定义输入 - 使用 el-select 配合 allow-create
      <el-select
        v-model="paramForm[param.name]"
        allow-create
        filterable
        default-first-option
      >
        <el-option v-for="opt in param.options" :value="Number(opt)" :label="String(opt)" />
      </el-select>
  else:
    // 无 options 的情况...
```

#### 【严重】问题4：options 数组元素类型未定义

**位置**：2.1.1 后端Schema

**问题分析**：
`options` 字段说明是 `string/int/float`，但没有规定是否允许混用，以及如何保证类型一致性。

**遗漏场景**：
- 如果 options 是 `[10, "auto", 100]`，前端 el-select 的 value 类型会不一致
- 如果 options 是 `["10", "20", "30"]`（全是字符串），但 type=int，是否需要自动转换？

**修复建议**：
1. 在前后端约定：options 数组内元素类型必须与 `type` 字段一致
2. Products.vue 提交时应做类型转换：
   ```javascript
   if (param.type === 'int') {
     param.options = param.options.map(v => parseInt(v, 10))
   } else if (param.type === 'float') {
     param.options = param.options.map(v => parseFloat(v))
   }
   ```
3. Commands.vue 读取时应确保类型一致：
   ```javascript
   if (param.type === 'int') {
     paramValue = parseInt(paramForm[param.name], 10)
   } else if (param.type === 'float') {
     paramValue = parseFloat(paramForm[param.name])
   }
   ```

#### 【严重】问题5：空字符串默认值被误转为0

**位置**：2.2.1 默认值类型严格匹配

**当前代码**：
```javascript
type=int/float：paramForm[name] = Number(default) || 0
```

**问题分析**：
- `Number('')` 返回 0，`0 || 0` 结果是 0，看似正确
- 但如果用户想表示"无默认值"（故意留空），会被错误转为 0
- 0 可能被误解为有效默认值

**修复建议**：
```javascript
if (param.type === 'int' || param.type === 'float') {
  if (default !== '' && default !== null && default !== undefined) {
    const num = param.type === 'int' ? Number(default) | 0 : Number(default)
    paramForm[name] = isNaN(num) ? '' : num
  } else {
    paramForm[name] = ''  // 空值保持空字符串
  }
}
```

#### 【中等】问题6：正则验证执行时机不明确

**位置**：2.2.2 伪代码

**问题分析**：
方案说"失焦时用正则验证"，但没有明确：
1. 验证失败时如何提示
2. 验证失败是否阻止提交
3. 是否需要在输入过程中也做实时校验

**修复建议**：
1. 添加实时校验（在 `el-input` 上使用 `validate-event` 和自定义规则）
2. 失焦时做一次完整校验
3. 提交前再次校验
4. 使用 `el-form-item` 的 `prop` 和 `rules` 机制：
   ```javascript
   const paramRules = {
     pattern: [
       { 
         validator: (rule, value, callback) => {
           if (!value) callback()
           try {
             new RegExp(value)
             callback()
           } catch (e) {
             callback(new Error('正则表达式无效'))
           }
         },
         trigger: 'blur'
       }
     ]
   }
   ```

#### 【中等】问题7：后端Schema验证缺失

**位置**：三、需修改的文件清单

**问题分析**：
后端 `schemas/__init__.py` 说无需改动，但新增字段没有任何类型验证。如果前端提交了非法数据，后端可能会静默接受或报错。

**修复建议**：
在后端添加可选字段验证（使用 Pydantic 的 Optional）：
```python
from typing import Optional, List, Union
from pydantic import BaseModel, Field

class ServiceParam(BaseModel):
    name: str
    type: str = Field(..., pattern="^(string|int|float|bool)$")
    required: bool = False
    default: Optional[Union[str, int, float, bool]] = None
    description: Optional[str] = ""
    options: Optional[List[Union[str, int, float]]] = []
    allow_custom: Optional[bool] = True
    min: Optional[Union[int, float]] = None
    max: Optional[Union[int, float]] = None
    step: Optional[Union[int, float]] = None
    pattern: Optional[str] = None
    
    class Config:
        extra = "allow"  # 允许额外字段，保持向后兼容
```

#### 【中等】问题8：Commands.vue 也需要 min <= max 校验

**位置**：六、风险点

**问题分析**：
风险点表格中"min > max 配置错误"只在 Products.vue 添加校验，但 Commands.vue 在用户输入时也应该实时校验。

**修复建议**：
在 Commands.vue 的 `el-input-number` 中，确保 min/max 属性在渲染时就保证正确：
```javascript
// 在渲染前校验并修正
const minVal = Number(param.min)
const maxVal = Number(param.max)
const safeMin = (isNaN(minVal) || (maxVal < minVal)) ? undefined : minVal
const safeMax = (isNaN(maxVal) || (maxVal < minVal)) ? undefined : maxVal
```

```javascript
  if (Array.isArray(service.input_params)) {
    // New format: [{name, type, required, default, description, options, allow_custom, min, max, step, pattern}, ...]
    currentServiceParams.value = service.input_params
    service.input_params.forEach(param => {
      // 类型安全的默认值处理
      if (param.default !== undefined && param.default !== '') {
        if (param.type === 'bool') {
          // bool 类型：兼容多种 truthy 值
          paramForm[param.name] = [true, 'true', 'True', 1, '1'].includes(param.default)
        } else if (param.type === 'int') {
          // int 类型：字符串转数字，空值保持空字符串
          const num = Number(param.default)
          paramForm[param.name] = isNaN(num) ? '' : num
        } else if (param.type === 'float') {
          // float 类型
          const num = parseFloat(param.default)
          paramForm[param.name] = isNaN(num) ? '' : num
        } else {
          // string 类型
          paramForm[param.name] = String(param.default)
        }
      } else {
        // 无默认值时的处理
        if (param.type === 'bool') {
          paramForm[param.name] = false
        } else if (param.type === 'int' || param.type === 'float') {
          paramForm[param.name] = ''
        } else {
          paramForm[param.name] = ''
        }
      }
      
      // 确保新字段有安全默认值（向后兼容）
      if (param.options === undefined) param.options = []
      if (param.allow_custom === undefined) param.allow_custom = true
      if (param.min === undefined) param.min = ''
      if (param.max === undefined) param.max = ''
      if (param.step === undefined) param.step = ''
      if (param.pattern === undefined) param.pattern = '^[a-zA-Z0-9_]+$'
    })
  } else if (typeof service.input_params === 'object') {
    // Legacy format: {"key": {type, description, ...}}
    const params = []
    Object.keys(service.input_params).forEach(key => {
      const def = service.input_params[key]
      const param = {
        name: key,
        type: typeof def === 'object' && def.type ? def.type : 'string',
        required: false,
        default: typeof def === 'object' && def.default !== undefined ? def.default : '',
        description: typeof def === 'object' && def.description ? def.description : key,
        options: [],
        allow_custom: true,
        min: '',
        max: '',
        step: '',
        pattern: '^[a-zA-Z0-9_]+$'
      }
      params.push(param)
      paramForm[param.name] = param.type === 'bool' ? false : ''
    })
    currentServiceParams.value = params
  }
}
```

**为何这样改**：
1. 对 bool/int/float 类型的默认值做严格类型转换
2. 空字符串默认值保持空字符串，不误转为 0
3. 对新字段添加安全默认值，确保向后兼容旧数据

```javascript
// 获取参数的安全 min 值（处理 NaN 和 min > max 的情况）
function getSafeMin(param) {
  if (param.min === '' || param.min === undefined || param.min === null) return undefined
  const minVal = Number(param.min)
  const maxVal = Number(param.max)
  if (isNaN(minVal)) return undefined
  if (!isNaN(maxVal) && maxVal < minVal) return undefined
  return minVal
}

// 获取参数的安全 max 值
function getSafeMax(param) {
  if (param.max === '' || param.max === undefined || param.max === null) return undefined
  const minVal = Number(param.min)
  const maxVal = Number(param.max)
  if (isNaN(maxVal)) return undefined
  if (!isNaN(minVal) && maxVal < minVal) return undefined
  return maxVal
}

// 获取参数的校验规则
function getParamRules(param) {
  const rules = []
  
  // required 校验
  if (param.required) {
    rules.push({
      required: true,
      message: `${param.name} is required`,
      trigger: 'blur'
    })
  }
  
  // string 类型的正则校验
  if (param.type === 'string' && param.pattern) {
    rules.push({
      validator: (rule, value, callback) => {
        if (!value) {
          callback()
          return
        }
        try {
          new RegExp(param.pattern)
          callback()
        } catch (e) {
          callback(new Error('Invalid pattern'))
        }
      },
      trigger: 'blur'
    })
  }
  
  return rules.length > 0 ? rules : null
}
```

**为何这样改**：
1. bool 类型继续使用 el-switch
2. string + 有 options：使用 el-select + allow-create + filterable 支持选择和自定义输入
3. string + 无 options：使用 el-input
4. int/float + 有 options + allow_custom=false：使用纯 el-select
5. int/float + 有 options + allow_custom=true：使用 el-select + allow-create
6. int/float + 无 options：使用 el-input-number 并设置 min/max/step
7. 添加 getSafeMin/getSafeMax 处理 min > max 的错误配置

```javascript
for (const param of currentServiceParams.value) {
  const value = paramForm[param.name]
  
  // 必填校验
  if (param.required && (value === undefined || value === '' || value === null)) {
    ElMessage.error(`${param.name} is required`)
    return
  }
  
  // 空值时跳过其他校验
  if (value === undefined || value === '' || value === null) continue
  
  // int/float 类型范围校验
  if (param.type === 'int' || param.type === 'float') {
    const numValue = Number(value)
    if (!isNaN(numValue)) {
      const minVal = Number(param.min)
      const maxVal = Number(param.max)
      if (!isNaN(minVal) && numValue < minVal) {
        ElMessage.error(`${param.name} value ${numValue} is less than min ${minVal}`)
        return
      }
      if (!isNaN(maxVal) && numValue > maxVal) {
        ElMessage.error(`${param.name} value ${numValue} exceeds max ${maxVal}`)
        return
      }
    }
  }
  
  // string 类型正则校验
  if (param.type === 'string' && param.pattern) {
    try {
      if (!new RegExp(param.pattern).test(value)) {
        ElMessage.error(`${param.name} does not match pattern ${param.pattern}`)

      // 正则表达式错误，跳过校验
    }
  }
  
  // allow_custom=false 时值必须在 options 中
  if (param.allow_custom === false && param.options?.length > 0) {
    const valueStr = String(value)
    const optionsStr = param.options.map(o => String(o))
    if (!optionsStr.includes(valueStr)) {
      ElMessage.error(`${param.name} must be one of the allowed values`)
      return
    }
  }
}

    }
  }
```
