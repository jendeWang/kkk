# 服务参数定义与命令下发功能整改方案

## 一、问题分析

### 问题 1：服务参数定义功能不足（Products.vue）

当前在产品编辑 → 服务标签 → 添加服务中，参数定义仅有以下字段：
- `name`（参数名）
- `type`（类型：string/int/float/bool）
- `required`（是否必填）
- `default`（默认值）
- `description`（描述）

**缺少的功能**：
1. **可选值（常用值）**：用户无法预设常用值列表，在下发页面点击输入框可弹出气泡选择
2. **是否允许手动输入**：无法控制是否允许用户自由输入，还是只能从可选值中选择
3. **范围限制**：数字类型参数缺少最小值/最大值范围限制
4. **正则限制**：字符串类型缺少正则表达式验证规则
5. **默认值类型处理**：当前 default 存为字符串，没有根据 type 做类型转换

### 问题 2：命令下发页面参数输入框无法输入（Commands.vue）

**问题根因分析**：
1. **响应式对象动态 key 问题**：`paramForm` 是 `reactive({})`，通过 `paramForm[param.name] = xxx` 动态赋值，在 Vue 3 中理论上是响应的，但在首次渲染时如果 key 不存在，可能导致输入框初始无法输入
2. **数字类型默认值问题**：`default` 在 Products.vue 中存为字符串，Commands.vue 中读出来也是字符串，配合 `v-model.number` 会造成类型不匹配的警告
3. **缺少参数编辑**：Products.vue 中只有「新增服务」功能，没有「编辑已有服务参数」的功能，已添加的服务无法修改参数定义

## 二、整改方案

### 2.1 参数定义数据结构升级

#### 2.1.1 后端 Schema（schemas/__init__.py）

**当前参数结构（数组形式）**：
```json
{
  "name": "brightness",
  "type": "int",
  "required": false,
  "default": "",
  "description": ""
}
```

**升级后的参数结构**：
```json
{
  "name": "brightness",
  "type": "int",
  "required": false,
  "default": 100,
  "description": "灯光亮度",
  "options": [10, 50, 100],
  "allow_custom": true,
  "min": 0,
  "max": 100,
  "pattern": "^[a-zA-Z0-9]+$",
  "step": 1
}
```

**字段说明**：
| 字段 | 类型 | 适用类型 | 说明 |
|------|------|----------|------|
| `name` | string | 全部 | 参数名 |
| `type` | string | 全部 | string/int/float/bool |
| `required` | bool | 全部 | 是否必填 |
| `default` | any | 全部 | 默认值（与 type 匹配） |
| `description` | string | 全部 | 参数描述 |
| `options` | array | string/int/float | 可选值列表（预设常用值），如 [10, 50, 100] |
| `allow_custom` | bool | string/int/float | 是否允许手动输入自定义值（false 则只能从 options 选） |
| `min` | number | int/float | 最小值（数字范围限制） |
| `max` | number | int/float | 最大值（数字范围限制） |
| `step` | number | int/float | 步进值（默认 1 或 0.01） |
| `pattern` | string | string | 正则表达式（默认 `^[a-zA-Z0-9_]+$`） |

**后端改动**：无需修改数据库字段（因为 input_params 是 JSON 字段），仅需更新文档注释说明新结构。

#### 2.1.2 前端 Products.vue 参数定义表单升级

修改 Products.vue 中 `addServiceParam()` 函数和参数编辑表单：

**改动点 A：`addServiceParam()` 初始化新字段**
```javascript
// 旧：
serviceParams.value.push({
  name: '', type: 'string', required: false, default: '', description: ''
})

// 新：
serviceParams.value.push({
  name: '',
  type: 'string',
  required: false,
  default: '',
  description: '',
  options: [],          // 新增：可选值列表
  allow_custom: true,   // 新增：是否允许手动输入
  min: '',              // 新增：数字最小值
  max: '',              // 新增：数字最大值
  step: '',             // 新增：数字步进
  pattern: '^[a-zA-Z0-9_]+$'  // 新增：字符串正则
})
```

**改动点 B：参数编辑表单扩展（根据 type 动态显示字段）**

在现有 5 个字段基础上，新增以下表单项目：
- **可选值（options）**：多标签输入框（el-input + tag 形式），用户输入逗号分隔或回车添加多个值
- **允许手动输入（allow_custom）**：el-switch 开关（默认开启）
- **最小值（min）**：仅 type=int/float 时显示
- **最大值（max）**：仅 type=int/float 时显示
- **步进值（step）**：仅 type=int/float 时显示
- **正则表达式（pattern）**：仅 type=string 时显示，默认值 `^[a-zA-Z0-9_]+$`

**改动点 C：参数默认值类型转换**
- 当 type=int 或 float 时，default 转为 number 类型存储（而非字符串）
- 当 type=bool 时，default 用 el-switch（true/false）

**改动点 D：编辑已有服务的功能**
- 在服务列表表格中增加「编辑」按钮
- 点击「编辑」时，将服务数据加载到 serviceForm 和 serviceParams 中，复用新增服务的对话框进行编辑
- 提交时调用 updateService API（需要后端提供）或 addService 的变种

### 2.2 命令下发页面升级（Commands.vue）

#### 2.2.1 参数输入框修复（解决无法输入的问题）

**核心修复：确保 paramForm 的响应式和默认值初始化正确**

1. **使用 shallowReactive + 显式 key 预注册**
   - 将 `paramForm` 改成在 `loadServiceParams()` 时先清空再重新赋值，确保所有 key 都在对象中

2. **默认值类型严格匹配**
   - type=int/float：`paramForm[name] = Number(default) || 0`
   - type=bool：`paramForm[name] = default === true || default === 'true'`
   - type=string：`paramForm[name] = String(default || '')`

3. **输入组件改进**
   - string 类型：如果设置了 options，则用 el-select + allow-create（可选择可输入）
   - int/float 类型：如果设置了 options，则用 el-select（可选值）+ 允许自定义输入
   - int/float 类型：应用 min/max/step 属性到输入框上
   - bool 类型：继续使用 el-switch

#### 2.2.2 根据新的参数字段渲染合适的输入控件

新的参数渲染逻辑（用伪代码表达）：

```
for each param in currentServiceParams:
  if param.type == 'bool':
    <el-switch v-model="paramForm[param.name]" />
  else if param.type == 'string':
    if param.options 有值:
      <el-select
        v-model="paramForm[param.name]"
        :allow-create="param.allow_custom"
        filterable
      >
        <el-option v-for="opt in param.options" :value="opt" :label="opt" />
      </el-select>
    else:
      <el-input
        v-model="paramForm[param.name]"
        :placeholder="param.description || param.name"
      />
      (若 param.pattern 有设置，失焦时用正则验证)
  else if param.type == 'int' or param.type == 'float':
    if param.options 有值 and !param.allow_custom:
      <el-select v-model="paramForm[param.name]">
        <el-option v-for="opt in param.options" :value="Number(opt)" :label="opt" />
      </el-select>
    else:
      <el-input-number
        v-model="paramForm[param.name]"
        :min="Number(param.min) || undefined"
        :max="Number(param.max) || undefined"
        :step="Number(param.step) || (param.type === 'int' ? 1 : 0.01)"
      />
      (若有 options，可在旁边放一个快捷选择气泡/按钮)
```

#### 2.2.3 参数提交前的验证增强

发送命令前，除了校验 required，还增加：
- 数字范围校验：`min <= value <= max`
- 字符串正则校验：`new RegExp(pattern).test(value)`
- allow_custom=false 时校验：值必须在 options 列表中

## 四、具体实施步骤

### Step 1：升级 Products.vue 参数定义表单

1. 修改 `addServiceParam()`，添加新字段初始化
2. 扩展参数表单 UI（在现有 `<div v-for="(param, index) in serviceParams">` 中新增以下表单项）：
   - `options`：用 el-input + 动态 tag 方式（类似 el-select 的 tag 模式），或简单用逗号分隔字符串，提交时 split 成数组
   - `allow_custom`：el-switch
   - `min` / `max` / `step`：条件显示（type 为 int/float 时）
   - `pattern`：条件显示（type 为 string 时），带默认值 `^[a-zA-Z0-9_]+$`
3. 提交时，根据 `type` 将 `default` 做类型转换（string → number/bool）
4. 新增「编辑服务」功能：在服务列表表格中增加「编辑」按钮，点击时将服务加载到对话框，编辑后提交更新

### Step 2：修复 Commands.vue 参数输入

1. 在 `loadServiceParams()` 中，确保 `paramForm` 的所有 key 都被显式赋值（先用 Object.keys 清空，再逐一赋值）
2. 将默认值做严格类型转换：int/float 用 `Number()`，bool 用 `=== true`，string 用 `String()`
3. 重写参数渲染逻辑，按上述 2.2.2 的规则选择合适的组件
4. 提交前增加范围、正则、options 校验

### Step 3：端到端测试

1. 登录系统，进入产品管理
2. 编辑一个产品，添加服务并定义参数（测试各种 type 的字段是否正确显示）
3. 设置默认值、可选值、范围等，保存
4. 进入命令下发页面，选择设备和刚设置的服务
5. 验证参数输入框是否正常渲染并可输入
6. 验证默认值是否自动填充
7. 验证可选值是否弹出选择
8. 验证范围/正则限制是否生效
9. 点击发送命令，验证设备模拟器是否收到参数

## 五、向后兼容说明

- 数据库中 input_params 字段是 JSON，新增字段对旧数据完全兼容（旧数据缺字段时，前端读取时设合理默认值：options=[]、allow_custom=true、min/max/step/pattern=undefined）
- 后端无需改动数据库迁移
- 旧的 Dict 格式参数继续兼容（Commands.vue 中已有的 legacy format 处理逻辑保留）

## 六、风险点

| 风险 | 应对方案 |
|------|----------|
| `paramForm` 动态 key 响应式失效 | 改用 `reactive({})` + 每次 `loadServiceParams` 时重新创建对象并赋值 |
| 默认值字符串转数字时 NaN | 使用 `Number(val) || 0` 或 `Number(val) ?? undefined`，并在 UI 上提示 |
| 正则表达式写得不对导致完全无法输入 | 在 pattern 输入框旁加「测试」按钮，或用 `try-catch` 包裹正则验证 |
| el-select + allow-create 的使用复杂 | 先用简单的「el-select + 无 options 时退化为 el-input」策略，后续再优化 |
